# Blueprint

1인 프로덕트 매니저를 위한 에이전트 스킬.
PRD, 디자인 시스템, 작업 아이템 등 설계 문서를 체계적으로 생성하고 관리합니다.

> **핵심 철학:** 이 스킬이 만드는 문서들은 에이전트에게 더 구체적이고 올바른 작업을 지시하는 데 도움을 줍니다. 다만 문서는 일부 변경에 취약할 수 있으며, 의도적으로 작업 이력을 남기는 것을 목표로 합니다. 문서는 SSOT(Single Source of Truth)가 아닙니다 — 코드가 항상 더 정확한 현재 상태를 나타냅니다.

## 주요 기능

### 1. 요구사항 분석

사용자의 아이디어나 요구사항을 PRD(제품 요구사항 문서)로 구체화합니다. 에이전트가 명확화 질문을 통해 빠진 부분을 채우고, 사용자의 검토·승인을 거쳐 PRD를 확정합니다.

> Ryan Carson의 [create-prd](https://github.com/snarktank/ai-dev-tasks/blob/main/create-prd.md) 프롬프트를 참고하였습니다.

### 2. 설계 문서 제공 및 확장

설계 문서들은 버전별 작업 아이템을 생성할 때 에이전트가 참고할 수 있는 보조 자료입니다. 문서의 양식은 정해져 있지 않으며, 사용자가 자유롭게 추가할 수 있도록 확장에 열려 있습니다. 작업 아이템 생성 시 에이전트와의 대화를 통해 참고할 설계 문서를 선택할 수 있습니다.

### 3. 소프트웨어 개발

개발은 한 번에 끝나지 않습니다. 에이전트에게 PRD와 필요에 따라 연계되는 설계 문서, 이전 버전의 작업 아이템 이력 등을 확인시켜 정교한 작업을 지속적으로 수행하세요.

당신의 요청이 기존 PRD에 없는 내용이거나, 다른 방향으로 전환될 수도 있습니다. 이때 PRD의 기존 내용은 삭제되지 않고 취소선으로 이력이 보존되며, 새로운 요구사항이 추가됩니다.

> Ryan Carson의 [generate-tasks](https://github.com/snarktank/ai-dev-tasks/blob/main/generate-tasks.md) 프롬프트를 참고하였습니다.

### 4. 멀티레포 지원

최대의 고민거리는 멀티레포 환경에서의 문서 관리였습니다. 단일 레포로 시작하되, 멀티레포 분리가 필요할 때 심볼릭 링크를 통해 원본 `.blueprint/` 폴더를 연결하세요. 어디서든 수정할 수 있으며, 각 프로젝트별로 작업 아이템이 버전별로 관리됩니다.

## 사용법

에이전트에게 자연어로 요청하거나 `/blueprint` 슬래시 명령어를 사용합니다.

### 기획 단계

트리거: 기획, PRD, 초안 등 | `/blueprint prd`

```
"todo 앱 기획해줘"
"이 요구사항으로 PRD 만들어줘"
```

에이전트가 프로젝트 루트에 `.blueprint/` 폴더를 생성하고, 사용자와 함께 PRD를 검토·승인합니다.

### 설계 단계

기획 이후 에이전트에게 더 구체적인 작업을 지시하려면, `.blueprint/(<scope>)/design` 폴더에 관련 문서들을 추가해주세요. 디자인 시스템, sitemap, 아키텍처, 배포 전략 등 형식에 제한 없이 자유롭게 첨부할 수 있습니다.

### 개발 단계

트리거: 개발, 구현, 구축, 작업 등 | `/blueprint dev`

```
"알림 기능 개발 시작하자"
"v0.2.0 작업 만들어줘"
```

PRD의 기능 요구사항(FR-ID)을 기반으로 작업 아이템을 생성하고, 승인 후 개발을 진행합니다.

### 멀티레포 연동

트리거: 문서 연동, 청사진 연동 등 | `/blueprint link`

```
"기존 청사진 연동해줘"
```

다른 레포의 `.blueprint/` 폴더를 심볼릭 링크로 연결하여, 여러 레포에서 동일한 설계 문서를 공유합니다.

## 생성되는 폴더 구조

```
(project-root)/
└── .blueprint/
    ├── prd-<기능명>.md              # 제품 요구사항 문서
    ├── (<scope>)/                   # 프로젝트별 그룹 폴더 (예: (todo-demo))
    │   ├── design/                  # 설계 문서
    │   │   ├── lofi-design-tokens.md
    │   │   ├── sitemap.md
    │   │   └── ...
    │   └── construction/            # 작업 아이템
    │       ├── v0.1.0-mvp.md
    │       ├── v0.2.0-feature-notification.md
    │       └── ...
    ├── README.md
    └── LICENSE
```

**scope**는 현재 레포/앱 이름을 소괄호로 감싼 그룹 폴더입니다. 멀티레포 환경에서는 하나의 `.blueprint/` 안에 여러 scope가 공존합니다.

## 주요 규칙

- **PRD는 승인 후 삭제하지 않는다** — 변경 시 취소선 처리하고 새 항목을 추가한다
- **작업 아이템은 승인 후 수정하지 않는다** — 요구사항이 바뀌면 새 작업 아이템을 생성한다
- **기능 요구사항은 FR-ID로 추적한다** — `FR-001`, `FR-002` 형식으로 채번하여 PRD와 작업 아이템을 연결한다
